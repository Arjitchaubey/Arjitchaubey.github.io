5-Stage Pipelined RISC-V Processor

Project Overview

This project involved architecting and implementing a 32-bit, 5-stage pipelined RISC-V processor from the ground up in Verilog. The design supports a key subset of the ISA including R, I, S, U, and J-type instructions, making it capable of running meaningful programs.

Core Features

  • Architecture: Engineered a modular design that includes dedicated hazard detection and control units to resolve data and control hazards, ensuring correct program execution and minimizing pipeline stalls.
  • Verification: Verified the processor's functional correctness by simulating its execution of a pre-compiled `program.hex` test file. The testbench monitored pipeline registers and outputs to confirm accurate instruction processing.
  • Implementation: Successfully synthesized and implemented the design on a Xilinx FPGA, achieving timing closure with a positive Worst Negative Slack (WNS) of 4.012 ns, confirming the design's robustness at the target frequency.

Challenges & Solutions

  • Challenge: Managing the complexity of a large, modular codebase. Understanding and integrating the numerous module interconnections in the top-level design was a significant task.
    Solution: I adopted a systematic approach by first planning all top-level connections and data paths on paper. This "blueprint" served as a clear guide during the Verilog coding phase, preventing integration errors and streamlining the process.
  • Challenge: The initial hazard control unit failed to correctly handle control hazards from jump and load instructions, causing the processor to fetch incorrect subsequent instructions.
    Solution: I re-architected the control logic to intentionally stall the pipeline for two clock cycles upon detecting a jump or branch instruction. This gave the Program Counter (PC) the necessary time to calculate and fetch the correct target address, ensuring program integrity.

CNN Accelerator & Golden Reference Verification

Project Overview

This project demonstrates a complete hardware design and verification flow, a critical skill in the semiconductor industry. I designed a Verilog-based hardware accelerator for CNN operations and rigorously tested it using a "golden reference" model written in Python.

Core Features

  • Python Reference Model: Developed a behavioral model in Python to perform Multiply-Accumulate (MAC) operations on MNIST image data. This script automatically generated "golden" reference test vectors, including feature maps and kernel values.
  • Hardware Design: Coded a CNN accelerator module in Verilog, with a focus on creating a high-throughput data path and an efficient FSM-based control unit.
  • Verification Framework: Created a Verilog testbench that read the test vectors from the Python model and fed them into the hardware design. The testbench then automatically compared the hardware's output against the golden reference, flagging any mismatches.
  • Result: Successfully validated the functional correctness of the Verilog MAC unit, ensuring its output was a bit-perfect match with the golden reference model for all test cases.

Challenges & Solutions

  • Challenge: The data flow logic within the convolution engine and output buffer was complex. Ensuring the hardware could correctly process streaming data without errors was difficult to conceptualize initially.
    Solution: I carefully studied the data flow requirements and planned the FSM control logic before implementation. This allowed me to build a robust state machine that managed the data path correctly.
  • Challenge: Verifying that the hardware was truly cycle-accurate and bit-perfect compared to the high-level Python model.
    Solution: I implemented a systematic, multi-stage verification strategy. First, I performed sanity checks with simple, self-created `.hex` files. Then, I used the Python script to generate complex, realistic test vectors from the MNIST dataset. By replacing the simple inputs with these "golden" inputs in my testbench, I could definitively prove the hardware's correctness.

Handwritten Digit Classifier with Audio Feedback

Project Overview

This project showcases my skills in machine learning and software integration. It served as the motivational basis for my hardware acceleration projects by demonstrating the computational demands of neural networks.

Core Features

  • Objective: To design and implement a machine learning model that classifies handwritten digits from the MNIST dataset using a CNN and reproduces the identified digit through audio output.
  • Architecture & Technology: Built a Convolutional Neural Network using TensorFlow/Keras, composed of multiple convolutional and pooling layers for feature extraction, and a dropout layer to prevent overfitting.
  • Performance: After training on 60,000 images from the MNIST dataset, the model achieved a test accuracy of 98.4%, demonstrating robust performance across various handwriting styles.
  • Key Feature - Accessibility: The system incorporates a Text-to-Speech (TTS) engine (using gTTS) that provides real-time voice feedback of the predicted digit, making the application accessible for visually impaired users and interactive for educational purposes.

Challenges & Solutions

  • Challenge: I encountered significant difficulties installing the correct versions of the TensorFlow library and its dependencies on my local machine.
    Solution: To avoid setup delays and ensure a stable environment, I migrated the entire project to Google Colab. This allowed me to leverage a pre-configured, cloud-based platform with all necessary libraries readily available.
  • Challenge: Due to system and time constraints, I could only run a limited number of training epochs, which capped the model's potential accuracy.
    Solution: Despite the constraints, I optimized the model architecture to achieve a high accuracy of 98.4% within the limited training window, demonstrating efficient model design.
  • Challenge: I wanted to add a unique feature that went beyond simple visual classification.
    Solution: I integrated the gTTS (Google Text-to-Speech) library to provide real-time audio feedback of the classified digit. This not only enhanced the project but also added a valuable accessibility feature for visually impaired users.
